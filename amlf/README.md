# ÆTHERIS METATRON LATTICE FRAMEWORK

## Description

A multi-layer, graph-structured cognitive framework that organizes, links, and routes intelligence through defined node schemas and overlap-driven relationships across domains within the Ætheris ecosystem.

## Purpose

This folder is the working home for AMLF inside the Ætheris Automation Technologies repository.

It is intended to hold:

- framework definition
- schemas
- node and link datasets
- governance rules
- interface contracts
- topology and traversal docs
- MCP and CLI implementation surfaces
- observability and tracing guidance

## Folder intent

- `architecture/` contains validation, traversal, MVP, and architecture notes
- `data/` contains AMLF node and link datasets
- `diagrams/` contains topology and flow references
- `docs/` contains system narratives and supporting reference material
- `examples/` contains use cases and test scenarios
- `governance/` contains review, audit, and behavioral boundary rules
- `hooks/` contains governance and validation hooks for Claude Code workflows
- `interfaces/` contains SEER, ODIN, and VALKYRIE contracts
- `mcp/` contains AMLF MCP server implementation
- `cli/` contains the AMLF operator CLI
- `observability/` contains Phoenix and OTel tracing setup
- `prompts/` contains bootstrap and working prompts
- `schema/` contains JSON schemas and typed contracts
- `services/` contains traversal, review, graph admin, and related services
- `skills/` contains Claude Code skills

## Working rules

- Treat AMLF as a framework first
- Keep governance mandatory
- Keep graph logic, protocol surface, and operator surface separated
- Use engineering language and traceable structures
